- **分层设计**：模块采用 Store 模式（`ProfileStore`, `GroupStore`, `WorkspaceStore`）封装数据访问，通过 `tokio::sync::Mutex` 保证异步环境下的状态一致性。
- **安全边界**：`profile.rs` 实现了元数据（JSON文件）与敏感数据（OS Keyring）分离存储的架构，并引入内存加密缓存（`PasswordCache`）以平衡安全性与用户体验（避免频繁触发系统授权）。
- **功能组件**：`monitor.rs` 基于 SSH exec 通道实现轻量级远程指标采集；`forward.rs` 利用 `russh` 底层通道能力实现本地、远程及动态 SOCKS5 端口转发，并通过 `Arc<Notify>` 实现优雅取消。
- **依赖方向**：所有 Store 组件均作为 Tauri State 注入，依赖于 `serde` 进行序列化，`dirs` 确定配置路径，以及 `keyring` 处理系统级密钥管理。
